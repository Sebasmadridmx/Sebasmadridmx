# Hi, I'm Sebastian 👋

I'm a senior hydraulic modeller based in Edinburgh, working with InfoWorks ICM
to model and analyse water and drainage networks across the UK.

My focus is on making hydraulic modelling workflows faster and less painful,
whether that's through ICM scripting, custom tooling, or bringing open geospatial
data into the modelling process.

---

## What I do

- **Hydraulic modelling** with InfoWorks ICM, from network build to results analysis
- **ICM scripting** using Ruby to automate repetitive tasks, batch process models,
  and generate reports
- **Web tools** built around ICM workflows, React frontends with Node.js backends
  that turn model outputs into something anyone on the team can use
- **Open data integration**, pulling in datasets like Scottish Public Sector LiDAR,
  OS MasterMap, and flood risk layers to support modelling decisions

---

## Stack

`InfoWorks ICM` `Ruby` `React` `Node.js` `Python` `QGIS` `GeoJSON` `WMS/WFS`

---

## Currently building

Small tools that sit alongside ICM and take the manual effort out of common tasks.
Things like survey card generation, model performance assessment, and geospatial
data exploration.
